### Configuration File Location

```bash

# Linux
~/.config/ElectronExecutor/config.json
```

### Common Configuration Commands

```bash
# Reset configuration to factory defaults
electron-cli config --reset

# Enable verbose logging for debugging
electron-cli config --set log_level=debug

# Change the default script directory
electron-cli config --set script_path="/home/user/my-scripts"

# Toggle auto-update on or off
electron-cli config --set auto_update=true

# Set maximum memory allocation (in MB)
electron-cli config --set max_memory=512

# Export your current configuration for backup
electron-cli config --export > my_backup_config.json

# Import a configuration from backup
electron-cli config --import my_backup_config.json
```

### Configuration Parameters Reference

| Parameter | Type | Default | Description |
|---|---|---|---|
| `log_level` | string | `info` | Logging verbosity: `debug`, `info`, `warn`, `error` |
| `auto_update` | boolean | `true` | Automatically download and install updates |
| `script_path` | string | `./scripts` | Default directory for script storage |
| `max_memory` | integer | `256` | Maximum RAM allocation in megabytes |
| `theme` | string | `dark` | UI theme: `dark`, `light`, `system` |
| `injection_delay` | integer | `0` | Delay in milliseconds before injection starts |
| `obfuscation` | boolean | `false` | Enable payload obfuscation |
| `telemetry` | boolean | `false` | Send anonymous usage statistics |
| `language` | string | `en` | Interface language code |
| `notification_sound` | boolean | `true` | Play sound on successful execution |

## 🔧 Troubleshooting & Common Issues

### ❌ Tool won't launch on Windows

**Symptoms:** Double-clicking the shortcut does nothing, or a brief flash appears and disappears.

**Solutions:**
1. Right-click and select **Run as Administrator**
2. Ensure your antivirus has not quarantined any files — add an exclusion for the installation folder
3. Install the latest Visual C++ Redistributable and .NET Runtime
4. Try running from the command line to see error output:
```bash
cd "C:\Program Files\ElectronExecutor"
.\ElectronExecutor.exe --verbose
```

### ❌ Injection fails or times out

**Symptoms:** The tool connects but scripts do not execute, or you see a timeout error.

**Solutions:**
1. Make sure the target application is fully loaded before attempting injection
2. Increase the `injection_delay` in your configuration to `3000` (3 seconds)
3. Disable any overlay software (screen recorders, FPS counters) temporarily
4. Restart both the tool and the target application

### ❌ Android APK won't install

**Symptoms:** "App not installed" or "Parse error" on Android.

**Solutions:**
1. Verify that **Unknown Sources** is enabled in your security settings
2. Ensure you have enough free storage (at least 200 MB)
3. Uninstall any previous version before installing the new electron executor apk
4. If using Android 12+, you may need to enable "Allow from this source" specifically for your browser

### ❌ macOS "damaged application" warning

**Symptoms:** macOS says the app is damaged and can't be opened.

**Solutions:**
1. Open Terminal and run:
```bash
xattr -cr /Applications/ElectronExecutor.app
```
2. Then try launching again from the Applications folder
3. If the issue persists, temporarily disable Gatekeeper:
```bash
sudo spctl --master-disable
```
4. Re-enable Gatekeeper after launching successfully

### ❌ Scripts load but nothing happens in popular titles

**Symptoms:** No errors, but the script has no visible effect.

**Solutions:**
1. Confirm the script is compatible with the current version of the target
2. Check the real-time console for silent errors or warnings
3. Some scripts require specific conditions (being in a certain location, menu, or state)
4. Try a simple test script first to verify the injection pipeline is working
